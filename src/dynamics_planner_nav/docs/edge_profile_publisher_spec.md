# 需求规格：机器人边缘轮廓发布节点（Robot Edge Profile Publisher）

> 目标读者：在 **Go2 狗端板载电脑**上实现/部署该节点的人或系统。
> 本节点把机器人当前姿态的真实物理体积，压缩成一圈"边缘半径"持续发布，供远端 PC 的导航/碰撞模块订阅使用。

---

## 1. 目标
以机器人 `base_link` 为中心、机头方向为 x 轴，按角度采样得到机器人**边缘到中心的距离**（极坐标轮廓）。该轮廓随关节姿态（站/趴/迈腿）实时变化。狗端**持续发布**，PC 端订阅后即可用 `clearance ≈ 障碍物到中心距离 − 边缘半径[障碍物方位角]` 做快速碰撞判断。

## 2. 运行位置与启动方式
- 运行在 **狗端板载电脑**。
- **开机自启**（systemd service 或随导航 bringup 一起 launch 均可）。
- 启动时做一次初始化（见 §5.1），之后持续运行。

## 3. 运行环境/依赖
- ROS 2（与狗端现有版本一致，例如 Humble）。
- Python：`rclpy`、`tf2_ros`、`numpy`。
- 计算凸包需要：`trimesh`、`scipy`、`pycollada`（仅启动时用一次）。
- 机器人描述：`go2.urdf` 及其引用的网格文件（`dae`），路径需在狗端可访问。

## 4. 输入
- **TF**：`base_link` → 各连杆坐标系的实时变换。需要狗端运行 `robot_state_publisher`（用 URDF + `/joint_states`）提供 TF。
  - 相关连杆：躯干 `base_link`，四条腿各段 `{FL,FR,RL,RR}_{hip,thigh,calf,calflower,calflower1,foot}`，以及 `Head_upper/Head_lower`（凡 URDF 中带 `<visual><mesh>` 的连杆都纳入）。
- **URDF**：解析每个连杆的 visual 网格文件、视觉偏移 `origin(xyz,rpy)`、`scale`。

## 5. 处理算法
### 5.1 启动初始化（只做一次，约 1~2 秒）
1. 解析 URDF，得到每个连杆的 (网格文件, 视觉偏移矩阵, 缩放)。
2. 加载每个网格，计算其 **3D 凸包顶点**（用 scipy `ConvexHull`），并应用视觉偏移与缩放，得到该连杆在**连杆坐标系**下的凸包点集（每连杆几十个点，全身合计几百~上万点）。
3. 缓存这些凸包点（运行期不再读网格）。

### 5.2 每个发布周期（建议 1~5 Hz，可配）
1. 对每个连杆，用 TF 查 `base_link`→该连杆的当前 4×4 位姿，把其凸包点变换到 `base_link` 系。
2. 投影到地面（取 x、y），相对中心 `(0,0)` 求每个点的极角 `atan2(y,x)` 与半径 `hypot(x,y)`。
3. 按角度均匀分桶（桶数 = 该档点数），每桶取**最大半径**（最外边缘）；空桶用相邻桶线性插值（周期性）。
4. 得到该档的边缘半径数组。

### 5.3 五个精细度档位（点数=角分辨率）
同一条轮廓按不同点数输出：

| 档位 | 点数 N | 角分辨率 |
|---|---|---|
| FINE（精细） | 200 | 1.8° |
| SEMI_FINE（略精细） | 150 | 2.4° |
| NORMAL（正常） | 100 | 3.6° |
| COARSE（粗糙） | 50 | 7.2° |
| POINT_MASS（质点） | 10 | 36° |

> 5 档可全部发布，或按参数只发其中若干档。

## 6. 输出（发布）
- 话题名（建议）：`/robot_edge_profile`（如分档发布可用 `/robot_edge_profile/fine` 等）。
- 消息类型：`std_msgs/msg/Float64MultiArray`。
- 数据格式（约定）：`[N, r_0, r_1, ..., r_{N-1}]`
  - 第 0 个元素是点数 `N`；其后是 N 个边缘半径（单位 m）。
  - 第 k 个半径对应角度 `angle_k = -π + 2π*(k+0.5)/N`（弧度，x 朝前=0，CCW 为正）。
- 发布频率：1~5 Hz（参数可配，默认 5 Hz）。

## 7. 坐标约定
- 参考系：`base_link`，原点在机器人中心。
- x 朝前（机头方向），y 朝左，CCW 为正。
- 半径=中心到机器人边缘的距离（m）。

## 8. 参数（建议可配）
- `publish_rate_hz`（默认 5）
- `modes`（默认 `[200,150,100,50,10]`，或选择要发布的档位）
- `urdf_path`、`base_frame`（默认 `base_link`）
- TF 等待/超时设置

## 9. 边界与健壮性
- 某连杆 TF 暂时查不到：跳过该连杆，用其余连杆计算（不崩溃）。
- 启动时 TF 还没好：等待并重试，直到拿到首帧再开始发布。
- 中心若落在体积外（理论不会）：该方向半径给 0。

## 10. 验收标准
1. 启动后 1~5 Hz 稳定发布，每周期计算耗时应 < 几毫秒（点数 10~200 对耗时影响极小）。
2. 改变机器人姿态（站/趴/迈腿），发布的边缘半径**随之实时变化**。
3. PC 端订阅能正确解析出 N 与 N 个半径，且 `angle_k` 约定一致。
4. 用边缘半径重建的轮廓与真实网格地面投影基本吻合（最大偏差可接受，例如 < 0.1 m）。

## 11. 备注（实现提示）
- 运行期只需 `numpy` 变换凸包点 + 分桶，无需再依赖 trimesh/网格；trimesh/scipy/pycollada 仅 §5.1 用一次。
- 若希望狗端依赖更轻，可在 PC 离线算好各连杆凸包点导出小文件，狗端只加载该文件 + numpy（则狗端无需 trimesh/scipy/网格）。
